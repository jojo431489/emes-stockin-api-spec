工單領退料作業

POST ERP → MES
issue.wo.item.process

ERP 通知 MES 工單領退料作業，用於同步工單用料資料、領退料單及明細。

基本資訊

API 名稱
issue.wo.item.process
方法名稱
IssueWoItemProcess
目標資料表
material_requisition / material_apply_basis / material_apply_detail
資料方向
ERP → MES
用途
工單領退料作業同步
授權模組
material（用料輕鬆管）
授權檢查
此 API 需要 material（用料輕鬆管）模組授權，若無授權將回傳錯誤。

請求參數（外層）

欄位類型必填說明
doc_noStringY單據編號
create_dateStringY建立日期
statusStringY作業狀態（1/2/3/4）
remarkStringN備註
wo_item_detailArrayY工單用料明細陣列

請求參數（wo_item_detail 明細）

欄位類型必填說明

wo_noStringY工單編號（製令編號）
item_typeStringY物料等級（0=成品/1=原料）
item_noStringY品號（物料編號）
qtyNumericY異動數量（必須 > 0）

input_datetimeStringN輸入時間
warehouse_noStringN庫別
location_noStringN儲位
lot_noStringN批號
expiry_dateStringN有效日期
remarkStringN備註
positive_negativeStringY正負向（-1=領料/其他=退料）

作業狀態說明

狀態碼說明數量計算
1發料新增demand_quantity + qty
2發料刪除demand_quantity - qty
3退料新增demand_quantity - qty
4退料刪除demand_quantity + qty
數量檢核
執行扣減作業（狀態 2、3）時，若 resultQty < 0 將拋出錯誤：
JAVA_0119_00057：需領用量不足，無法執行扣減作業

處理邏輯

業務流程
  1. 檢查 material 模組授權
  2. 驗證 status 必須為 1/2/3/4
  3. 驗證 wo_no（製令編號）存在於 MODETAIL
  4. 查詢 material_requisition 取得目前 demand_quantity
  5. 依據 status 計算新的 resultQty
  6. 新增 material_requisition 記錄
  7. 新增/更新 material_apply_basis（領退料單頭）
  8. 新增 material_apply_detail（領退料單身）

資料庫更新 SQL

1. 檢查製令是否存在

SELECT 1 FROM MODETAIL WITH (NOLOCK) WHERE CMOID = :wo_no

2. 查詢目前需領用量

SELECT TOP 1 ISNULL(demand_quantity, 0) AS demand_quantity
FROM material_requisition WITH (NOLOCK)
WHERE modetail_cmoid = :wo_no
  AND material_id = :item_no
ORDER BY serial DESC

3. INSERT material_requisition（工單用料維護單身）

INSERT INTO material_requisition (
    serial, modetail_cmoid, material_type, material_id,
    opid, erp_wsid, material_batch, remark,
    demand_quantity, withdrawn_quantity, mr01
) VALUES (
    :serial,           -- MAX(serial) + 1
    :wo_no,            -- 工單編號
    :item_type,        -- 物料類型 (1=成品/2=原料)
    :item_no,          -- 品號
    :op_no,            -- 製程代號
    '',                -- 工作站代號
    :lot_no,           -- 批號
    :remark,           -- 備註
    :resultQty,        -- 計算後的需領用量
    0,                 -- 已領用量
    N'{當前時間}'       -- 建立時間
)

4. INSERT material_apply_basis（工單領退料單頭）

IF NOT EXISTS (
    SELECT 1 FROM material_apply_basis
    WHERE material_order_id = :doc_no
)
INSERT INTO material_apply_basis (
    material_order_id, material_order_type, factory_id,
    order_source, material_order_status,
    creator, create_time, modifier, modify_time
) VALUES (
    :doc_no,           -- 單據編號
    :positive_negative, -- 1=領料/2=退料
    '001',             -- 廠別
    '2',               -- 來源 (ERP)
    '1',               -- 狀態
    'ERP_SYSTEM',
    N'{當前時間}',
    'ERP_SYSTEM',
    N'{當前時間}'
)

5. INSERT material_apply_detail（工單領退料單身）

IF NOT EXISTS (
    SELECT 1 FROM material_apply_detail
    WHERE material_order_id = :doc_no
      AND material_order_seq = :seq
)
INSERT INTO material_apply_detail (
    material_order_id, material_order_seq, mo_id, material_seq,
    material_id, apply_qty, receive_qty, op_id, ws_id,
    material_type, standard_quantity,
    creator, create_time, modifier, modify_time
) VALUES (
    :doc_no,           -- 單據編號
    :seq,              -- 項次
    :wo_no,            -- 工單編號
    :serial,           -- 用料流水號
    :item_no,          -- 品號
    :qty,              -- 申請數量
    :qty,              -- 領用數量
    :op_no,            -- 製程代號
    '',                -- 工作站代號
    :item_type,        -- 物料類型
    :qty,              -- 標準數量
    'ERP_SYSTEM',
    N'{當前時間}',
    'ERP_SYSTEM',
    N'{當前時間}'
)

錯誤代碼

代碼訊息說明

JS_0097_00005製令編號不得為空值wo_no 為空
JAVA_0119_00058物料等級不得為空且必須為0或1item_type 驗證失敗
JS_0085_00004品號不得為空值item_no 為空
JAVA_0119_00060異動數量必須大於0qty 驗證失敗
JSP_0119_00009製令不存在MODETAIL 中查無此製令
JAVA_0119_00057需領用量不足，無法執行扣減作業扣減後數量為負數
eMES_Material_007用料輕鬆管模塊未授權 material 模組
