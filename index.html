1. API 基本資訊

1.1 拋送託外進貨單 API

建立委外收貨單資料

Outsourcing Purchase Stock-In Create

outsourcing.purchase.stockin.create
DSMS API: api-2062/vid-4462
項目說明
API 中文名稱建立委外收貨單資料（託外進貨單）
API 英文名稱Outsourcing Purchase Stock-In Create
CROSS API Codeoutsourcing.purchase.stockin.create
MES taskNameoutsourcingPurchaseStockinCreate
ESB 路徑/outsourcingPurchaseStockinCreate
API 用途說明MES 委外站出站後，將委外進貨資料拋送至 ERP，產生委外進貨單據
呼叫方向MES → ERP
觸發條件TRANSTYPE = "59"（委外進貨單）
呼叫時機1. 委外站出站完成時
2. 驗收入庫時
主要業務情境委外加工完成後，將成品入庫至 ERP 指定倉庫，更新委外工單狀態
同步/非同步同步呼叫
交易控管需要
備註回傳 TB001/TB002 回寫至 TL011/TL012

1.2 取消託外進貨單 API

委外採購收貨還原

Outsourcing Process (Cancel)

outsourcing.process
DSMS API: api-2058/vid-4571
項目說明
API 中文名稱委外採購收貨還原（取消託外進貨單）
API 英文名稱Outsourcing Process
CROSS API Codeoutsourcing.process
MES taskNameoutsourcingProcess
ESB 路徑/outsourcingProcess
API 用途說明取消已產生的 ERP 委外進貨單，還原庫存與委外工單狀態
呼叫方向MES → ERP
觸發條件TRANSORDERTYPE = "59"（委外單）
呼叫時機1. MES 執行取消出站
2. 品質異常需退回時
主要業務情境取消委外入庫，還原庫存與工單狀態
同步/非同步同步呼叫
交易控管需要
source_type 說明0=出貨還原, 1=回貨還原, 2=退貨還原

2.3 前置條件與限制條件

建立委外進貨單前置條件

條件說明
TRANSTYPE = 59必須為委外進貨單類型
TO013 = 1才會送 ERP
明細資料存在SFT_TRANSORDER_LINE_GRID 不可空
委外工單存在ERP 委外工單必須存在
數量合理入庫數量 > 0

取消委外進貨單前置條件

條件說明
TL011/TL012 有值必須有 ERP 單據編號
TRANSORDERTYPE = 59委外單才用此 API
進貨單未過帳已過帳無法取消
無後續異動未被其他單據引用
重要提醒：Race Condition 風險
取消出站時，若 ERP 尚未回傳進貨單號（TL011/TL012 為空），將無法正確呼叫取消 API。

程式碼位置：GeneralUpdater.java:1571 
if(!SftTransorderLine.getTl011().equals("") && !SftTransorderLine.getTl012().equals("")){
    // 只有 TL011/TL012 都有值時才呼叫取消 API
}

6. 錯誤代碼設計

錯誤代碼錯誤訊息說明發生情境
E001必填欄位缺失必填欄位未傳入Request 缺少必要欄位
E002委外工單不存在工單號在 ERP 查無資料MOTYPE-MONO 錯誤
E003委外工單狀態不允許工單已結案或取消工單狀態異常
E004數量必須大於零qty 小於等於 0數量錯誤
E005數量超過餘量超過委外工單可入庫量超量入庫
E101委外進貨單不存在取消時找不到原單outsource_no 錯誤
E102委外進貨單已過帳已完成過帳嘗試取消已過帳單據
E103委外進貨單已引用有後續單據庫存已被使用
E104庫存不足取消後庫存會變負庫存已耗用
E999系統錯誤ERP 內部錯誤Timeout、連線異常

8. 測試案例

8.1 建立委外進貨單測試案例

TC-C-001 正向測試
測試項目建立委外進貨單成功
前置條件1. TRANSTYPE = "59"
2. 委外工單存在且狀態正常
3. 委外站已出站
測試步驟1. MES 執行委外站出站
2. 觸發 outsourcingPurchaseStockinCreate
3. 檢查 Response
預期結果1. code = 200
2. TB001/TB002 有值
3. TL011/TL012 正確回寫
TC-C-002 負向測試
測試項目委外工單不存在
前置條件傳入不存在的 moType-moCode
測試步驟執行出站並呼叫 API
預期結果1. code 不等於 200
2. ISOK = false

8.2 取消委外進貨單測試案例

TC-P-001 正向測試
測試項目取消委外進貨單成功
前置條件1. TL011/TL012 有值
2. TRANSORDERTYPE = "59"
3. 進貨單未過帳
測試步驟1. MES 執行取消出站
2. 觸發 outsourcingProcess
3. 傳入 source_type = "0"
預期結果1. ISOK = true
2. ERP 庫存還原
3. 進貨單狀態變為已取消
TC-P-002 負向測試
測試項目TL011/TL012 為空不呼叫 API
前置條件TL011 或 TL012 為空值
測試步驟執行取消出站
預期結果不呼叫 outsourcingProcess API

9. 驗收標準

9.1 功能正確性驗收

驗收項目驗收條件驗收方式
資料正確性 1. MES 傳送的委外工單號、品號、數量正確寫入 ERP
2. ERP TB001/TB002 正確回寫至 TL011/TL012
3. 庫存數量正確增減 		比對 MES TRANSORDER_LINE 與 ERP 進貨單
API 路由正確性 1. TRANSTYPE=59 使用 outsourcingPurchaseStockinCreate
2. TRANSORDERTYPE=59 使用 outsourcingProcess 		檢查 LOG 中的 taskName

9.2 系統穩定性驗收

驗收項目驗收條件
呼叫成功率正常情境 大於等於 99%
回應時間小於等於 5 秒（不含網路）
Timeout 處理正確記錄 LOG，不產生髒資料
