1. API 基本資訊

1.1 抛送入庫單 API

工單完工入庫單資料新增

Work Order Stock-In Data Create

stockin.data.create
項目說明
API 中文名稱工單完工入庫單資料新增
API 英文名稱Work Order Stock-In Data Create
API Codestockin.data.create
對應 ESB 服務/outsourcingPurchaseStockinCreate (TRANSTYPE=59)
/sendTransForm (其他)
API 用途說明MES 工單完工後，將入庫資料拋送至 ERP，由 ERP 產生完工入庫單據
呼叫方向MES → ERP
呼叫時機1. 工單最後一站出站完成時
2. 批次完工入庫作業時
3. 手動觸發入庫時
主要業務情境製造完工後，將成品入庫至 ERP 指定倉庫，更新 ERP 庫存數量與工單狀態
同步/非同步同步呼叫
交易控管需要 MES 與 ERP 需確保資料一致性
備註回傳 TB001/TB002 為 ERP 產生的單據類別/單號，需回寫至 SFT_TRANSORDER_LINE.TL011/TL012

1.2 取消倉管入庫 API

工單完工入庫還原

Work Order Stock-In Data Process (Cancel)

stockin.data.process
項目說明
API 中文名稱工單完工入庫還原
API 英文名稱Work Order Stock-In Data Process (Cancel)
API Codestockin.data.process
對應 ESB 服務/outsourcingProcess (TRANSORDERTYPE=59)
/cancelStockIn (其他)
API 用途說明取消已產生的 ERP 入庫單，還原庫存數量與工單狀態
呼叫方向MES → ERP
呼叫時機1. MES 執行取消出站作業時
2. 品質異常需退回重工時
主要業務情境取消已完成的入庫作業，ERP 需還原庫存扣減並作廢原入庫單據
同步/非同步同步呼叫
交易控管需要
備註取消前需檢查 TL011/TL012 是否有值，有值才呼叫 ERP

2.4 前置條件與限制條件

✅ 新增入庫前置條件

條件說明
工單存在ERP 工單必須存在且狀態允許入庫
途程完成MES 最後一站已完成出站
數量合理入庫數量 > 0 且 ≤ 工單可入庫餘量
TRANSORDER_LINE明細資料必須存在且有效
TO013 檢查TO013=1 時才送 ERP

⚠️ 取消入庫前置條件

條件說明
TL011/TL012 有值必須有 ERP 回傳的單據編號
入庫單存在原入庫單號必須存在於 ERP
狀態允許入庫單未過帳或未確認
無後續異動入庫單未被其他單據引用
⚠️ 重要提醒：Race Condition 風險
取消出站時，若 ERP 尚未回傳入庫單號（TL011/TL012 為空），將無法正確呼叫取消 API。 建議在取消出站前，先確認 ERP 回傳已完成。

參考案例：CLAUDE.md 中 2026-03-12 記錄的「0000675600 取消出站 outsourcingProcess 未呼叫問題」

錯誤代碼錯誤訊息說明發生情境

TC-C-001 正向測試
測試項目新增入庫成功
前置條件1. 工單 5101-20260319001 存在且狀態為生產中
2. 最後站已出站
3. 倉庫 FG01 有效
測試步驟1. MES 執行出站作業
2. 觸發 TransOrderDataToJson.doErp()

2. 回傳有效的 TB001/TB002

3. Msg 包含錯誤訊息

9. 驗收標準

9.1 功能正確性驗收

驗收項目驗收條件驗收方式
資料正確性 1. MES 傳送的工單號、品號、數量正確寫入 ERP
2. ERP 產生的 TB001/TB002 正確回寫至 TL011/TL012
3. 庫存數量正確增減 		1. 比對 MES TRANSORDER_LINE 與 ERP 入庫單內容
2. 驗證 ERP 庫存交易明細
邏輯正確性 1. TRANSTYPE=59 正確路由至 outsourcingPurchaseStockinCreate
2. TRANSORDERTYPE=59 正確路由至 outsourcingProcess
3. 取消時正確還原庫存 		1. 檢查 LOG 中的 taskName
2. 執行入庫/取消前後庫存比對
檢核正確性 1. TL011/TL012 為空時不呼叫取消 API
2. TO013≠1 時不送 ERP
3. 錯誤時 ISOK=false 		1. 執行所有負面測試案例
2. 驗證 LOG 和回傳值

9.2 系統穩定性驗收

驗收項目驗收條件驗收方式
呼叫成功率 正常情境下 API 呼叫成功率 ≥ 99% 連續執行 100 次正常請求，統計成功次數
回應時間 單筆入庫回應時間 ≤ 5 秒（不含網路延遲） 記錄 API 處理時間，計算平均值
Timeout 處理 Timeout 時正確記錄 LOG，不產生髒資料 模擬 Timeout 情境，檢查 ERPReturnErrorRec.log

9.3 可追蹤性驗收

驗收項目驗收條件驗收方式
LOG 記錄 1. 送出單據類型和內容有記錄
2. ERP 回應有記錄
3. 錯誤時記錄 Exception 類型 		檢查 logs/ERPReturnErrorRec.log
格式：[ERP_SEND_ERROR] taskName=xxx, errorType=xxx
單據追蹤 1. 可由 MES 批號(LOTNO)追蹤到 ERP 入庫單號
2. TL011/TL012 正確對應 ERP 單據 		查詢 SFT_TRANSORDER_LINE 的 TL011/TL012
對比 ERP 入庫單

9.4 測試覆蓋性驗收

驗收項目驗收條件
正向測試✅ 新增入庫成功（一般/委外）

負向測試✅ 工單不存在
