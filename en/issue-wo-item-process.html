Work Order Material Issue/Return

POST ERP → MES
issue.wo.item.process

ERP notifies MES of work order material issue/return operations, synchronizing material usage data, issue/return orders and details.

Basic Information

API Name
issue.wo.item.process
Method Name
IssueWoItemProcess
Target Tables
material_requisition / material_apply_basis / material_apply_detail
Data Direction
ERP → MES
Purpose
Work order material issue/return sync
Required Module
material (Material Management)
Authorization Check
This API requires the material module license. An error will be returned if not authorized.

Operation Status

1Issue - Createdemand_quantity + qty
2Issue - Deletedemand_quantity - qty
3Return - Createdemand_quantity - qty
4Return - Deletedemand_quantity + qty
Quantity Validation
When performing deduction operations (status 2, 3), if resultQty < 0, an error will be thrown:
JAVA_0119_00057: Insufficient demand quantity, cannot perform deduction

Processing Logic

Business Flow
  1. Check material module authorization
  2. Validate status must be 1/2/3/4
  3. Validate wo_no exists in MODETAIL
  4. Query material_requisition for current demand_quantity
  5. Calculate new resultQty based on status
  6. Insert material_requisition record
  7. Insert/Update material_apply_basis (issue/return header)
  8. Insert material_apply_detail (issue/return detail)
